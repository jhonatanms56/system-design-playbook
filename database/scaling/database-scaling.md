# Database Scaling

As data volume and traffic grow, databases must scale to maintain performance.

## Vertical Scaling (Scaling Up)

Increasing CPU, RAM, or storage on a single machine. Limited by hardware capacity.

## Horizontal Scaling (Scaling Out)

### Read Replicas
Offloading read traffic to secondary nodes.

### Sharding (Horizontal Partitioning)
Distributing data across multiple database instances based on a shard key.

### Partitioning (Vertical/Horizontal within a DB)
Dividing large tables into smaller, more manageable pieces.
